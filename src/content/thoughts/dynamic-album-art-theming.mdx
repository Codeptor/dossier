Aurora is a Spicetify theme built around a single trick: every accent color in the UI traces back to the current album art. When the track changes, the whole interface shifts hue.

The pipeline is small. When a track plays, fetch the artwork URL from `Spicetify.Player.data` (rewriting `spotify:image:` URIs to `https://i.scdn.co/image/`). Decode the bitmap to a canvas, sample pixels into 16 hue bins. Filter out near-greys (`saturation < 0.15`) and near-blacks/whites (`lightness < 0.1` or `> 0.9`) so a moody cover doesn't lock the UI to charcoal. Pick the dominant bin, return HSL. Convert back to RGB hex when applying — CSS variables propagate the rest.

```ts
const HUE_BINS = 16;
const SAT_MIN = 0.15;
const LIGHT_MIN = 0.1;
const LIGHT_MAX = 0.9;
```

The result drives a small set of variables (`--accent`, `--accent-hover`, glow tint), which the rest of the CSS leans on. Glass-morphism surfaces with `backdrop-filter: blur()` make the wash feel ambient instead of garish — the album art also drifts behind the entire app at low opacity, so the accent color reads as belonging to the *space*, not painted on it.

Two extras worth mentioning. The right panel has Queue / Recent / Friends tabs with a docked player up top — drag-reorderable rows, which Spicetify doesn't ship by default. The lyrics view runs a 5-line focus through AMLL (word-level), lrclib, or Spotify's own provider in fallback order; click any line to seek, `[`/`]` nudges timing ±50ms.

Cache the accent per album art URL. Hue extraction on every track change is otherwise wasteful — most playlists have ten unique covers across a hundred tracks.
